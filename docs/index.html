Правила GitHub от Роскомнадзора

Обязательно к исполнению всеми разработчиками

1

Запрещены мемы в коммитах

Каждый commit message должен содержать только официальную информацию. Смайлики, мемы и сарказм — под запретом. "Fixed bug" — да. "Fixed this stupid bug finally" — нет.

2

Код должен быть предсказуемым

Запрещены неожиданные side effects, "магические" числа и код, который работает, но никто не понимает почему. Если код работает — это подозрительно.

3

README обязателен на русском

Каждый репозиторий должен содержать README.md на русском языке. README на английском допускается только как приложение к основному README.

4

Запрещены VPN и прокси

Любой код, связанный с обходом блокировок, VPN, прокси или TOR — немедленно удаляется. Исключение: код для блокировки VPN.

5

Только одобренные зависимости

Все npm/pip/cargo зависимости должны быть из одобренного реестра. Запрещены пакеты с названиями типа "is-odd", "left-pad" или "true".

6

Эмодзи в коде — под запретом

Использование эмодзи в именах переменных, функций и комментариях запрещено. Переменная "const pizza = true" будет немедленно отмечена как опасный контент.

Часто задаваемые вопросы

Ответы на самые популярные вопросы от разработчиков

Почему мой репозиторий заблокирован, если я просто учусь программировать?

Обучение — это хорошо, но обучение правильному программированию — ещё лучше. Ваш репозиторий "hello-world" содержал файл README.md на английском языке без русского перевода. Это грубое нарушение правила #3.

Можно ли использовать GitHub Copilot?

Copilot разрешён, но все сгенерированные предложения проходят дополнительную проверку. Если Copilot предложит использовать VPN или саркастичный комментарий — мы немедленно удалим этот фрагмент кода и отправим вам предупреждение.

Что делать, если я случайно запушил мем в репозиторий?

Немедленно сделайте git revert и git push --force-with-lease. Затем напишите официальное письмо в Роскомнадзор с извинениями и приложите справку о прохождении курса "Как не делать мемы". Штраф составит 1000 рублей за каждый мем.

Можно ли форкать заблокированные репозитории?

Нет. Форк заблокированного репозитория автоматически помечается как "зеркало опасного контента". Вместо этого мы рекомендуем форкать только официальные репозитории Роскомнадзора, которые содержат одобренные шаблоны кода.

Как часто проверяются репозитории?

Проверка происходит в режиме реального времени. Каждый git push, pull request и даже сохранение черновика немедленно анализируется нашими алгоритмами. Среднее время реакции — 0.3 секунды, что быстрее, чем вы успеете сказать "git commit -m fix".
